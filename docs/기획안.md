# 디지털디톡스 앱 기획안 (Android 전용)

> 한 줄 요약: 사용자가 직접 복잡하게 관리하지 않아도, **캐릭터 코치**가 휴대폰 사용시간에 따라 칭찬하거나 경고하며 디지털디톡스를 유도하는 **Android 전용 캐릭터 코칭 앱**.

---

## 1. 프로젝트 개요
사용자가 휴대폰 사용시간을 줄이고 자연스럽게 디지털디톡스를 실천하도록 돕는 Android 전용 앱이다.
기존 습관 관리 앱처럼 많은 기록 입력이나 복잡한 설정을 요구하지 않는다. 앱을 설치하고 확인하기만 해도, **캐릭터 코치**가 사용자의 휴대폰 사용시간 상태를 알려주고 상황에 맞는 멘트로 디지털디톡스를 유도한다.
전체 구조는 매우 단순하게 구성한다. 사용자가 할 행동은 최소화하고, 앱이 먼저 현재 상태를 알려주는 방식으로 설계한다.

## 2. 앱 핵심 콘셉트
**설치만 해도 관리받는 디지털디톡스 앱.**
앱을 켜면 오늘의 휴대폰 사용시간 · 목표시간 · 현재 상태를 바로 확인할 수 있다. 별도 기록 입력 없이 캐릭터가 사용시간을 기준으로 코칭한다.
가장 중요한 요소는 **캐릭터**다. 단순한 장식이 아니라 디지털디톡스를 이끌어주는 트레이너이자 코치다.
- 사용시간이 적을 때 → 칭찬 / 응원
- 사용시간이 많을 때 → 걱정 / 경고
- 모드에 따라 → 팩폭(T) 또는 다정한 위로(F)

## 3. 서비스 목표
- 사용자가 자신의 휴대폰 사용시간을 쉽게 인식하게 한다.
- 사용시간이 길어질수록 자연스럽게 경각심을 느끼게 한다.
- 캐릭터 멘트로 디지털디톡스를 가볍고 친근하게 실천하도록 유도한다.
- 복잡한 기능 없이 홈 화면만 봐도 현재 상태를 이해하게 한다.
- T/F 코칭 모드로 사용자 성향에 맞는 말투를 선택하게 한다.

## 4. 타깃 사용자
강제 차단 앱이나 복잡한 습관 앱은 부담스러워하지만 사용시간은 줄이고 싶은 사용자.
- 사용시간이 많은 걸 알지만 쉽게 줄이지 못하는 사용자
- 디지털디톡스 의지가 오래가지 않는 사용자
- 귀여운 캐릭터/말투 기반 동기부여에 반응하는 사용자
- 강한 팩폭이 필요한 사용자 / 부드러운 위로가 필요한 사용자
- 복잡한 설정 없이 간단한 앱을 원하는 사용자

## 5. 플랫폼 기준
- **Android 전용.** iOS 대응 없음.
- 사용시간 데이터는 Android Usage Stats 연동을 고려해 설계한다.
- 초기 단계에서 실제 기기 연동이 어려우면 **임시 데이터**로 화면·흐름을 먼저 구현한다.
- 중요한 것은 데이터 연동 여부보다, **사용시간에 따라 캐릭터 상태·멘트가 달라지는 경험**을 먼저 완성하는 것.
- 온보딩에서 **개인정보 수집 동의 + Usage Stats 권한 동의**를 받고 시작한다.

## 6. 정보 구조
하단 네비게이션은 **2개만** 사용한다: `홈`, `기록`.

**만들지 않는 화면:** 로그인 · 회원가입 · 마이페이지 · 설정 페이지 · 커뮤니티 · 랭킹 · 친구 기능.
사용자는 홈과 기록 화면만 오가며 디지털디톡스 상태를 확인한다.

## 7. 주요 기능
### 7-1. 오늘 휴대폰 사용시간 확인
홈에서 오늘 사용시간을 크게 표시. 예) `오늘 사용시간 / 3시간 24분`.

### 7-2. 목표 시간 설정
하루 목표 사용시간 설정. **기본값 3시간.** 옵션: 1·2·3·4·5시간 / 직접 입력.
별도 설정 페이지 없이 **홈 화면 안에서** 바로 수정.

### 7-3. 목표 대비 상태 확인
예) `목표보다 42분 적게 사용 중` (긍정), `목표보다 1시간 12분 초과` (경고).

### 7-4. T/F 코칭 모드 선택
핵심 차별점. 홈에서 코칭 말투 선택.
- **T모드**: 직설적·현실적 팩폭 코칭
- **F모드**: 다정·부드러운 위로형 코칭

> MBTI T/F에서 아이디어를 가져왔지만 MBTI 검사 앱은 아니다. 성향/기분에 맞춰 말투를 고르는 기능.

> 멘트는 **3개 톤(일반 / 사용시간 많음 / 위험 단계)** 으로 작성하고, 5단계에 매핑한다. (매핑표는 10장)

## 8. T 끄숑 멘트 (팩폭형)
트레이너 톤. **적게 쓰면 인정·칭찬, 많이 쓸수록 팩폭 강도↑.**
특징: 직설적 · 현실적 · 군더더기 없음 · 팩폭 · 트레이너 느낌.

**일반(적게 사용 · 인정형)**
- 좋습니다. 오늘 페이스 깔끔하네요.
- 지금 관리 상태, 합격입니다.
- 이 정도면 잘 하고 있는 겁니다.
- 오늘은 휴대폰한테 안 졌네요.
- 딱 좋습니다. 이대로 유지하세요.
- 군더더기 없는 사용량입니다.
- 오늘 자제력 좋습니다.
- 스크롤 욕구, 잘 참고 있네요.
- 이 페이스면 끄숑이도 만족입니다.
- 휴대폰 주도권, 지금은 회원님한테 있습니다.

**사용시간 많음(팩폭)**
- 하루가 휴대폰 속으로 사라졌습니다.
- 이 정도면 디지털 거주자입니다.
- 눈 건강이 도망가는 중.
- 액정이 회원님 얼굴을 기억하겠습니다.
- 휴대폰 없이 10분 버틸 수 있나요?
- 사용시간이 조금 심각합니다.
- 지금 필요한 건 충전기가 아니라 휴식입니다.
- 오늘도 현실보다 화면을 더 많이 봤네요.
- 스크롤 국가대표 출전 가능.
- 엄지손가락만 성장 중입니다.
- 오늘도 액정을 열심히 키우셨네요.
- 휴대폰이 직업입니까?
- 손가락만 운동 중.
- 현실 접속은 언제 하시나요?
- 또 스크롤 하셨네요.
- 오늘도 알고리즘의 승리.
- 눈도 소모품입니다.
- 휴대폰은 쉬는데 회원님은 안 쉬네요.
- 액정과 사랑에 빠지셨군요.
- 휴대폰이 회원님을 사용 중입니다.

**위험 단계**
- 회원님은 휴대폰과 동거 중입니다.
- 이제 그만 보세요.
- 눈이 울고 있습니다.
- 시력은 복구권이 없습니다.
- 오늘은 진짜 많이 봤습니다.
- 잠시 내려놓으세요.
- 액정인간 단계 진입.
- 끄숑도 말리기 포기 직전입니다.

## 9. F 끄슝 멘트 (위로형)
따뜻하게 위로·응원한다. 비난하지 않고 부드럽게 쉬어가도록 유도.
특징: 다정함 · 위로 · 응원 · 부드러운 권유 · 부담 없는 말투.

**일반**
- 오늘도 고생 많았어요.
- 많이 바쁜 하루였나 봐요.
- 잠시 눈을 쉬게 해주세요.
- 천천히 줄여도 괜찮아요.
- 끄숑은 회원님이 걱정돼요.
- 오늘 하루는 어땠나요?
- 쉬어가는 시간도 필요해요.
- 물 한 잔 마시고 올까요?
- 눈도 잠시 휴식이 필요해요.
- 지금도 충분히 잘하고 있어요.

**사용시간 많음**
- 오늘은 휴대폰과 함께한 시간이 길었네요.
- 많이 지친 하루였나 봐요.
- 무언가에 기대고 싶었던 하루였을까요?
- 잠시 창밖을 바라보는 건 어떨까요?
- 회원님의 눈 건강이 걱정돼요.
- 조금만 쉬어도 몸이 훨씬 편해질 거예요.
- 내일은 조금 덜 봐도 괜찮아요.
- 오늘도 잘 버텨냈어요.
- 휴대폰 말고도 좋은 것들이 많답니다.
- 끄숑이 응원할게요.

**위험 단계**
- 오늘은 정말 오래 사용하셨네요.
- 충분히 쉬어야 할 것 같아요.
- 눈도 마음도 휴식이 필요해요.
- 오늘만큼은 일찍 쉬어보는 건 어떨까요?
- 내일은 조금 더 가볍게 보내봐요.
- 회원님의 건강이 가장 중요해요.
- 끄숑은 회원님이 오래 건강했으면 좋겠어요.

## 10. 목표 대비 캐릭터 상태 (5단계)

> **판정 기준: 목표 대비 사용 비율 `r = 사용시간 / 목표시간`.** (2026-06-15 변경)
> 절대 사용시간이 아니라 **사용자가 정한 목표 대비**로 단계를 매겨, 목표 상태와 캐릭터 코칭이 항상 같은 방향으로 움직이게 한다. `r = 1.0`(목표 도달) 지점에서 "목표 초과" 경고와 캐릭터 경고 단계가 동시에 켜진다. (배경·논의는 `개선과제.md` 참고)

| 단계 | 목표 대비 비율 | 상태명 | 캐릭터 느낌 |
|---|---|---|---|
| 1 | ~50% 미만 | 아주 좋은 흐름 | 활기참, 웃는 표정, 칭찬 |
| 2 | 50 ~ 100% (목표 이내) | 괜찮은 편 | 평온, 가벼운 주의 |
| 3 | 100 ~ 125% (목표 막 초과) | 주의 필요 | 걱정, 경고 시작 |
| 4 | 125 ~ 175% | 사용시간 많음 | 진지, 잔소리, 경고 |
| 5 | 175% 이상 | 디톡스 필요 | 강한 경고, 속상함 |

### 멘트 톤 ↔ 5단계 매핑
멘트는 3개 톤(일반 / 사용시간 많음 / 위험 단계)으로 작성하고, 5단계에 아래와 같이 매핑한다. 같은 톤 안에서는 멘트를 랜덤/순환으로 보여준다.

| 단계 | 상태명 | 멘트 톤 |
|---|---|---|
| 1 | 아주 좋은 흐름 | 일반 |
| 2 | 괜찮은 편 | 일반 |
| 3 | 주의 필요 | 사용시간 많음 |
| 4 | 사용시간 많음 | 사용시간 많음 |
| 5 | 디톡스 필요 | 위험 단계 |

> 전체 멘트 목록은 **8장(T 끄슝) · 9장(F 끄슝)** 참고. T모드는 팩폭형, F모드는 위로형으로 동일 단계에서 말투만 달라진다.

## 11. 홈 화면 기획
중심 화면. 캐릭터가 먼저 보이고 그 아래 사용시간·목표 정보가 정리된다.
**구성 순서:** 상단 타이틀 → 캐릭터 → 말풍선 → 오늘 사용시간 → 목표시간 → T/F 모드 → 현재 상태.

- **상단**: 짧은 문구. 예) “오늘의 디지털디톡스 / 휴대폰과 조금만 거리두기 해볼까요?”
- **캐릭터**: 가장 크게. 상태별 감정 표현(웃음/평온/걱정/혼냄/지침). 이미지 교체 쉬운 구조, 초기엔 placeholder 가능.
- **말풍선**: 현재 사용시간 + 선택된 T/F 모드 멘트. 실제로 말 거는 느낌.
- **오늘 사용시간**: 큰 숫자. 단, 캐릭터보다 위에 오지 않음.
- **목표시간**: 표시 + `목표 수정` 버튼(홈 안에서 변경).
- **목표 대비 상태**: 적게 사용(긍정) / 초과(경고).
- **T/F 모드 선택**: 토글, 선택된 모드 시각 강조.
- **현재 상태**: 짧은 라벨(아주 좋은 흐름 / 괜찮은 편 / 주의 필요 / 사용시간 많음 / 디톡스 필요), 캐릭터 감정과 연결.

## 12~15. 기록 화면 기획
복잡한 통계가 아니라 일간/월간 사용시간을 간단히 확인하는 화면.
**구성 순서:** 상단 타이틀 → 일간/월간 선택 → 총 사용시간 카드 → 이전 기간 대비 카드 → 세로 막대 그래프.

- **일간**: 최근 7일. 오늘 총 사용시간 / 전일 대비(증가·감소) / 요일별 세로 막대 그래프.
- **월간**: 최근 6개월. 이번 달 총 사용시간 / 전월 대비(증가·감소) / 월별 세로 막대 그래프.

## 16. 디자인 방향
- Android 모바일 앱처럼 보이게. 화면 기준 **390px**, 하단 네비 고정.
- 분위기: 귀엽되 유치하지 않게 · 차분하고 정돈됨 · 신뢰감 · 캐릭터 중심 친근함.
- 카드형 UI 중심 · 넓은 여백 · 짧고 명확한 정보.

## 19. 화면별 핵심 우선순위
- **홈**: 캐릭터 → 멘트 → 오늘 사용시간 → 목표시간 → T/F 모드 → 현재 상태. (캐릭터가 코칭하는 느낌이 가장 중요)
- **기록**: 일간/월간 선택 → 총 사용시간 → 이전 기간 대비 → 세로 막대 그래프. (가볍게 흐름 확인)

## 20. 개발 시 반영 방향 / 최종 제작 요구사항
- Android 모바일 앱 기준, 개인정보 동의 + Usage Stats 권한 동의 화면 포함.
- 홈/기록 2개 화면, 하단 네비 홈/기록만.
- 홈: 캐릭터 최상단 크게 + 상태별 말풍선 멘트 + 오늘 사용시간 큰 숫자 + 목표시간/수정 버튼 + 목표 대비 상태 + T/F 모드 버튼 + 현재 상태 라벨.
- T = 직설/팩폭, F = 다정/위로. 사용시간 적을수록 긍정, 길수록 경고/혼냄.
- 기록: 일간(오늘 총·전일 대비·최근 7일 막대) / 월간(이번 달 총·전월 대비·최근 6개월 막대).
- UI는 단순·깔끔. 캐릭터 중심 디지털디톡스 앱 인상 분명히. 복잡한 기능 추가 금지. 출시 목표 완성도.
- 초기엔 실제 Usage Stats를 완전히 연결하지 않아도 됨. 사용시간에 따른 캐릭터 상태·멘트 변화 흐름이 잘 보이는 것이 핵심.

## 21. 한 줄 요약
사용자가 직접 복잡하게 관리하지 않아도, 캐릭터 코치가 휴대폰 사용시간에 따라 칭찬하거나 경고하며 디지털디톡스를 유도하는 **Android 전용 캐릭터 코칭 앱**.
