## ⭐ 핵심 결정 — ✅ 적용 완료 (2026-06-15)

**캐릭터 상태(5단계)를 무엇 기준으로 판정할 것인가? → (B) 목표 대비 비율 채택**

- 이전: `levelOf()`가 **절대 사용시간(시)** 기준 (0~2h=1단계 … 7h+=5단계).
- 문제였던 점: 목표시간은 사용자가 바꿀 수 있는데 캐릭터 단계는 목표와 무관 → "목표 초과(경고)"인데 캐릭터는 1단계(좋은 흐름)인 모순.
- **채택안 (B): 목표 대비 사용 비율 `r = 사용시간 / 목표시간` 단일 기준.**
  - `r = 1.0`(목표 도달) 지점에서 "목표 초과" 경고와 캐릭터 경고 단계가 **동시에** 켜짐 → 목표 상태와 캐릭터가 절대 따로 놀지 않음.
  - 목표를 바꾸면 캐릭터 상태도 즉시 함께 재계산됨.

| 비율(목표 대비) | 단계 | 상태명 | 멘트 톤 |
|---|---|---|---|
| ~50% 미만 | 1 | 아주 좋은 흐름 | 일반 |
| 50 ~ 100% | 2 | 괜찮은 편 | 일반 |
| 100 ~ 125% | 3 | 주의 필요 (목표 막 초과) | 사용시간 많음 |
| 125 ~ 175% | 4 | 사용시간 많음 | 사용시간 많음 |
| 175% 이상 | 5 | 디톡스 필요 | 위험 |

- 구현: `app.js`의 `STATE_BANDS` / `ratioOf()` / `levelOf(min, goal)` / `energyOf(min, goal)`. 컨디션 게이지(%)도 같은 구간에 묶여 단계와 항상 일치.

## 1. 온보딩 / 권한 흐름 — 🟡 부분 개선 (2026-06-15)

- **개선된 점**: 온보딩을 **3장 캐러셀**(① 환영 → ② 작동 방식(칭찬/걱정/T·F) → ③ 권한 동의)로 개편. 진행 점·다음 버튼·건너뛰기·좌우 스와이프 지원. 마지막 장에서 필수 동의 2개를 체크해야 시작 가능. (`app.js` 온보딩 캐러셀 컨트롤러, `index.html` `#onboarding`)
- **아직 남은 문제**: `onboarded=true`(`app.js:13`)가 저장되면 **다시 진입 불가**, 권한 **거부·재동의 경로 없음**.
- **개선 방향(잔여)**:
  - 권한 미동의/거부 상태를 구분해 안내 화면 유지
  - 설정성 재진입 경로(또는 권한 안내 배너)
  - 프로토타입에서는 "온보딩 다시 보기" 디버그 진입점이라도 확보

## 2. 목표 초과와 캐릭터 단계가 분리됨 — ✅ 해결 (2026-06-15)

- **이전 현상**: `levelOf()`는 절대시간, 목표대비 상태(`goalStatus`)는 따로 계산 → "목표 넘었어요" 경고가 떠도 캐릭터는 웃고 있는 모순.
- **해결**: 위 ⭐ 핵심 결정(B)대로 캐릭터 단계·말풍선 톤·컨디션 게이지·목표 상태를 모두 `r = 사용시간/목표` 단일 기준으로 통일. `r=1.0`에서 함께 경고로 전환.

## 3. 기록 화면 통계 지표 — ✅ 해결 (2026-06-15)

- **이전 현상**: 하단 통계가 *일/월 평균* + **"가장 적게 쓴 날/달"** → 디톡스 앱인데 경각심 유발과 거리가 멈.
- **해결**: **"가장 많은 날/달"** 로 변경(가장 많이 쓴 기간을 보여줘 주의 환기). (`app.js` `renderRecord`, `high` 인덱스)
- **추가 검토(선택)**: 더 나아가 "이번 주 목표 달성 N일" 류 성취 지표도 향후 고려 가능.

## 4. 사용시간이 고정 임시값 (데모 시연성)

- **현상**: `usage` 기본값 **135분(=2시간 15분) 하드코딩**, 화면에서 바꿀 방법 없음. (`app.js:5`)
- **왜 문제**: 단계별(1~5단계) 캐릭터·멘트 변화가 이 앱의 핵심인데, 데모에서 그 변화를 보여줄 수 없음.
- **개선 방향**: 데모용 슬라이더/버튼으로 사용시간을 조절해 단계 전환을 시연 가능하게. (출시 빌드에선 숨김)
- ⚠️ **주의(혼동 방지)**: 이건 "데모 시연"용일 뿐, **실제 사용시간 측정과는 별개**다. 진짜 측정은 아래 6번(네이티브 연동) 과제다.

## 5. 멘트가 날짜 시드로 하루 고정

- **현상**: `daySeed()`로 같은 날은 같은 멘트, 클릭해도 안 바뀜. (`app.js:165`)
- **왜 문제**: 의도된 설계지만("그날 하루는 고정"), 기획서의 "실제로 말 거는 느낌"과는 다소 거리. 재방문 시 신선함 약함.
- **개선 방향**: 하루 고정 유지 vs 방문/시간대마다 순환 — 컨셉 측면에서 한 번 재검토.

## 6. 실제 사용시간 측정(데이터 연동) 부재 — ⛔ 미해결 (플랫폼 제약)

- **현상**: 앱이 보여주는 사용시간은 OS가 잰 값이 아니라 **하드코딩된 placeholder(135분)**. 사용자가 실제 쓴 시간이 누적되지 않음.
- **왜 그런가**: 현재는 **브라우저 웹 프로토타입**이라 기기 전체 사용량을 읽을 권한이 원천적으로 없음. 실제 측정은 **Android 네이티브 앱 + Usage Stats(사용정보 접근 권한)** 으로만 가능. (온보딩 3장의 권한 동의가 이를 위한 것)
- **개선 방향(실 출시)**:
  - 안드로이드 앱으로 빌드 → Usage Stats 권한 → 측정값을 `store.usage`(`app.js:5` 자리)에 주입
  - 일별/월별 합계를 `dayHistory`/`monthHistory`(`seedHistory`, `app.js:100`)의 샘플 시드 대신 실제 값으로 채움
  - 현재 코드는 이 지점만 갈아끼우면 비교·그래프 로직이 그대로 동작하도록 설계돼 있음 (`app.js`의 `recordToday`/`buildSeries` 주석 참고)
- **참고**: 화면·코칭 흐름 완성이 먼저라는 기획 방침(기획안 5번)에 따른 의도된 현재 상태.

## 📌 최근 디자인 변경 이력 (2026-06-15, 프로토타입)

> 과제 항목은 아니지만 팀이 알아야 할 UI 변경. 코드: `index.html`, `styles.css`, `app.js`.

- **온보딩 3장 캐러셀** 신규 (위 1번 참고).
- **홈 하단 카드 통합**: 분리돼 있던 "오늘의 상태" + "오늘 사용시간" 2개 박스를 **하나의 글래스 카드**로 합침. 구성: `표정+이름+컨디션%` → `목표 대비 문구 + 게이지` → 구분선 → **오늘/목표/남은(초과) 3분할**. (예시이미지2의 3분할 + 기존 시안의 컨디션·표정 장점 결합)
  - 남은 시간 칸은 목표 초과 시 **"남은 시간 → 초과 시간"** 으로 라벨·색(녹색→빨강) 자동 전환.
  - 군더더기였던 일러스트 슬롯(deco1/deco2) 제거.
- **글씨 가독성**: 흰 카드 위에서 흐리던 연한 초록/노랑 글씨를 위해 단계별 **글씨 전용 진한색 `--accent-ink`** 도입(색상 코딩 유지, 명도↑).
- **전체 글씨 크기 +1단계** 상향.
- **폰트 교체**: Pretendard → **페이퍼로지(Paperlogy)**. 분위기를 더 포근하게. 라이선스 **SIL OFL = 상업적 이용 가능**(폰트 자체 판매만 금지), 굵기 100~900 전 지원. CDN: `fonts-archive/Paperlogy`.
- **말풍선**: 한국어 **단어(어절) 단위 줄바꿈**(`word-break:keep-all`) 적용, 위치를 캐릭터 머리 위 상단으로 조정(`.bubble-wrap` margin-top).
