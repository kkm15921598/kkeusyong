# 끄숑 · 기술 스택 (Tech Stack)

> 현재 끄숑 프로토타입이 **실제로 사용하는 기술**과 **아직 쓰지 않는 후보 기술**을 정리한 문서.
> (작성 기준: 2026-06-19, 저장소 실제 파일 기반)

---

## 한 줄 요약

끄숑은 현재 **순수 HTML + CSS + JavaScript + localStorage**로 만든 모바일 화면 프로토타입이다.
빌드 도구·프레임워크 없이 `node server.js`로 바로 미리보기하고, `docs/` 폴더로 GitHub Pages에 배포한다.

---

## 실제로 쓰고 있는 것 ✅

| 영역 | 실제 사용 |
|------|-----------|
| **마크업** | 순수 HTML (`index.html`) |
| **스타일** | 순수 CSS (프레임워크 없음) |
| **로직** | 순수 JavaScript (`app.js`, 번들러·모듈 시스템 없음) |
| **상태 저장** | 브라우저 `localStorage` (서버 DB 없음) |
| **캐릭터 표정** | 인라인 SVG로 직접 작성 (`FACES` 배열) |
| **로컬 서버** | Node.js 기본 `http` 모듈 직접 작성 (`server.js`, 포트 5173) |
| **문서 뷰어** | `marked.js` CDN 1줄 (기획안 `.md` 렌더용) |
| **호스팅** | `docs/` 폴더 → **GitHub Pages** 구조 |
| **개발 도구** | **Claude Code** (코드 작성 + 커밋) |
| **버전 관리** | Git / GitHub |

---

## 아직 쓰지 않는 것 ❌ (도입 후보)

| 기술 | 현재 상태 | 비고 |
|------|-----------|------|
| **Next.js** | 미사용 | `package.json`·`next.config` 없음 |
| **TypeScript** | 미사용 | 전부 `.js` |
| **Tailwind CSS** | 미사용 | CSS 직접 작성 |
| **PWA / PWA Builder** | 미사용 | `manifest.json`·service worker 없음 |
| **Vercel** | 미사용 | 배포는 GitHub Pages 방식 |
| **Figma** | 저장소엔 없음 | 단, `assets/mockups/`에 시안 이미지(`홈화면 구성 ver1/ver2.png` 등) 존재 |

---

## 앞으로 "진짜 앱"으로 만들 때 (제안)

프로토타입을 정식 제품으로 키운다면 다음 조합이 자연스럽다.

- **Next.js + TypeScript** — 화면 구조·라우팅·타입 안정성
- **Tailwind CSS** — 빠른 스타일링 (현재 직접 작성한 CSS를 점진 이전)
- **PWA** — 모바일 홈 화면 설치·오프라인 (디지털디톡스 앱 성격상 적합)
- **Vercel** — Next.js 배포 (현재 GitHub Pages에서 이전)
- **Figma** — 디자인 시안을 정식 디자인 시스템으로 관리

> 단계적 마이그레이션 권장: 현재 화면 로직(`app.js`)을 컴포넌트로 옮기고, localStorage 상태를 그대로 살린 뒤 프레임워크를 입히는 순서.
